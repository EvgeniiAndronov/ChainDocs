# 🚀 ChainDocs Demo - Быстрый старт

## 3 простых шага

### 1️⃣ Подготовка окружения

```bash
./demo/demo-prepare.sh
```

**Что делает:**
- ✅ Очищает старое окружение
- ✅ Собирает проект (сервер, клиенты, keygen)
- ✅ Генерирует ключи для 3 клиентов
- ✅ Создаёт тестовый документ

**Время:** ~30 секунд

---

### 2️⃣ Запуск демонстрации

```bash
./demo/demo-start.sh
```

**Что делает:**
- ✅ Запускает сервер на порту 8080
- ✅ Регистрирует ключи клиентов
- ✅ Запускает 3 клиента-демона с P2P
- ✅ Загружает тестовый документ
- ✅ Ожидает консенсус (3/2 подписи)

**Время:** ~15 секунд

**Результат:**
```
✅ Демонстрационная среда запущена!
📊 Статус:
  - Сервер: http://localhost:8080
  - 3 клиента (P2P порты: 9001, 9002, 9003)
  - Консенсус: 3/2 подписи
```

---

### 3️⃣ Остановка

```bash
./demo/demo-stop.sh
```

**Что делает:**
- ✅ Останавливает всех клиентов
- ✅ Останавливает сервер

**Время:** ~2 секунды

---

## 📝 Пример сессии

```bash
# 1. Подготовка
./demo/demo-prepare.sh

# 2. Запуск
./demo/demo-start.sh

# 3. Проверка (в другом терминале)
curl -s http://localhost:8080/api/blocks/last | jq

# 4. Загрузка нового документа
curl -X POST http://localhost:8080/api/upload \
  -F "file=@my-document.pdf"

# 5. Проверка через 5 секунд
curl -s http://localhost:8080/api/blocks/last | jq \
  '{height, signatures: (.signatures | length)}'

# 6. Остановка
./demo/demo-stop.sh
```

---

## 🔄 Повторный запуск

Если нужно запустить заново:

```bash
# Вариант A: Полная очистка
./demo/demo-prepare.sh  # Подготовка заново
./demo/demo-start.sh    # Запуск

# Вариант B: Быстрый перезапуск
./demo/demo-stop.sh     # Остановка
./demo/demo-start.sh    # Запуск (без очистки)
```

## 📊 Проверка работы

### Проверка блока

```bash
curl -s http://localhost:8080/api/blocks/last | jq
```

### Проверка консенсуса

```bash
curl -s http://localhost:8080/api/blocks/last/consensus | jq
```

### Проверка P2P подключений

```bash
grep "P2P" demo/demo_logs/client1.log | tail -10
```

**Ожидаемый вывод:**
```
✅ P2P server listening on localhost:9001
✅ P2P node started on localhost:9001
📢 Signature broadcasted via P2P
```

## ⚠️ Troubleshooting

### Порт 8080 занят

```bash
lsof -i :8080
kill <PID>
```

### P2P порты заняты

```bash
lsof -i :9001
lsof -i :9002
lsof -i :9003
kill <PID>
```

### Клиенты не подключаются

Проверьте логи:
```bash
tail -f demo/demo_logs/client1.log | grep P2P
```
