# 📘 РУКОВОДСТВО ПО РАБОТЕ С БЛОКЧЕЙН-СИСТЕМОЙ ДОКУМЕНООБОРОТА

## 🏗 ОБЩАЯ АРХИТЕКТУРА

```
[Сервер] <---> [Клиент 1] <---> [Клиент 2] <---> [Клиент N]
     ↑              ↑              ↑               ↑
     └── хранит      └── подписывают блоки
         блоки           своими ключами
```

## 📋 ПОЛНЫЙ ЦИКЛ РАБОТЫ

### **Шаг 1: Запуск сервера**
Сервер хранит блокчейн, принимает файлы и координирует подписи.

```bash
# Очистить всё перед началом (опционально)
make clean

# Запустить сервер
make run
```
**Ожидаемый вывод:**
```
🚀 Server starting on :8080
✅ Genesis block created: 9aade631...
```

Сервер доступен по адресу: `http://localhost:8080`

---

### **Шаг 2: Генерация ключей клиента**
Каждый клиент должен иметь свою пару ключей.

```bash
# Сгенерировать ключ
make keygen PASSWORD=mysecretpassword OUT=mykey.enc
```

**Ожидаемый вывод:**
```
✅ Key pair generated successfully!
📁 Private key (encrypted): mykey.enc
🔑 Public key: 5d1cb17def07135434c50169817062037fcfbf420ca9c9bf5eade6c8f39cf155
```

**Что создалось:**
- `mykey.enc` - зашифрованный приватный ключ (хранить в секрете!)
- Публичный ключ (показан на экране) - нужно зарегистрировать на сервере

---

### **Шаг 3: Регистрация публичного ключа на сервере**
Сервер должен "знать" клиента, чтобы принимать от него подписи.

```bash
# Зарегистрировать ключ (скопируйте ваш публичный ключ из предыдущего шага)
curl -X POST http://localhost:8080/api/register \
  -H "Content-Type: application/json" \
  -d '{"public_key":"5d1cb17def07135434c50169817062037fcfbf420ca9c9bf5eade6c8f39cf155"}'
```

**Ожидаемый ответ:**
```json
{
  "status": "registered",
  "key": "5d1cb17def07135434c50169817062037fcfbf420ca9c9bf5eade6c8f39cf155"
}
```

В логах сервера появится:
```
✅ Public key registered: 5d1cb17def07...
```

---

### **Шаг 4: Загрузка документа (автоматически создает блок)**

```bash
# Загрузить PDF файл
curl -X POST http://localhost:8080/api/upload \
  -F "file=@document.pdf"
```

**Ожидаемый ответ:**
```json
{
  "hash": "8760399b2376c9ff3e5c42e93db3020cf817ce82da1021066bbdf9a63aead371",
  "filename": "document.pdf",
  "size": 10038289,
  "uploaded": "2026-02-16T16:05:11.194729+03:00",
  "block_hash": "70318ca9892a3f532b832d4a16779d9182ca726f8269e385c137091ec108b26d"
}
```

**Что произошло:**
1. Файл сохранен в папку `./uploads/` под именем `8760399b...pdf`
2. Вычислен SHA-256 хэш файла
3. Создан новый блок с этим хэшем
4. Блок пока **без подписи** (ждет клиентов)

---

### **Шаг 5: Проверка созданного блока**

```bash
# Посмотреть все блоки
curl http://localhost:8080/api/blocks | json_pp

# Или только последний блок
curl http://localhost:8080/api/blocks/last | json_pp

# Или конкретный блок по хэшу

```

**Пример вывода:**

```json
   "timestamp" : "2026-02-16T13:05:11.184684Z"

```

### **Шаг 6: Запуск клиента для подписи блока**

Клиент в режиме **oneshot** проверит последний блок и подпишет его:

```bash
# Запустить клиента
make client PASSWORD=mysecretpassword MODE=oneshot
```

**Ожидаемый вывод:**
```
🔑 Loading private key...
✅ Key loaded. Public key: 5d1cb17def071354...
🔄 Running in oneshot mode
📦 Last block: height=1, hash=70318ca9
✍️  Signing block...
✅ Signature created: a72d117d5a70fffd...
🎉 Block 1 signed successfully!
```

### **Шаг 8: Проверка документа**

```bash
# Скачать документ по его хэшу
curl -OJ http://localhost:8080/api/documents/8760399b2376c9ff3e5c42e93db3020cf817ce82da1021066bbdf9a63aead371

# Проверить, что хэш совпадает
shasum -a 256 document.pdf
# Должно выдать: 8760399b2376c9ff3e5c42e93db3020cf817ce82da1021066bbdf9a63aead371
```

## ⚠️ ВАЖНЫЕ МОМЕНТЫ

### **1. Приватные ключи**
- `mykey.enc` - НИКОМУ НЕ ПОКАЗЫВАТЬ!
- Хранить в безопасном месте
- Без пароля не расшифровать

### **2. Публичные ключи**
- Можно показывать всем
- Обязательно регистрировать на сервере
- Без регистрации сервер отвергнет подпись

### **3. Документы**
- Хранятся в папке `uploads/`
- В блокчейне только их хэши
- Один документ = один блок

### **4. Подписи**
- Один блок могут подписать несколько клиентов (консенсус)
- Каждая подпись проверяется сервером
- Без подписи блок не считается подтвержденным

## 🚀 ПОЛНЫЙ ТЕСТОВЫЙ СЦЕНАРИЙ

Скопируйте и выполните последовательно:

```bash
# 1. Очистить и запустить сервер
make clean
make run
# (в другом терминале)

# 2. Сгенерировать ключ
make keygen PASSWORD=test123 OUT=test.key
# Сохраните публичный ключ из вывода

# 3. Зарегистрировать ключ
curl -X POST http://localhost:8080/api/register \
  -H "Content-Type: application/json" \
  -d '{"public_key":"<ВСТАВЬТЕ_ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ>"}'

# 4. Создать тестовый PDF
echo "Test document content" > test.pdf

# 5. Загрузить PDF
curl -X POST http://localhost:8080/api/upload \
  -F "file=@test.pdf"

# 6. Проверить блок (должен быть без подписи)
curl http://localhost:8080/api/blocks/last | json_pp

# 7. Подписать блок
make client PASSWORD=test123 MODE=oneshot

# 8. Проверить блок (теперь с подписью)
curl http://localhost:8080/api/blocks/last | json_pp

# 9. Скачать и проверить документ
curl -OJ http://localhost:8080/api/documents/<ХЭШ_ИЗ_ШАГА_5>
shasum -a 256 test.pdf
```
